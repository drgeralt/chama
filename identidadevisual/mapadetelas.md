Mapeamento de Telas e Fluxos de Navegação - Sistema ChamaEste documento descreve todas as telas do sistema Chama, seus componentes principais, o comportamento dependente do papel do utilizador (Role) e o fluxo de navegação entre elas. Ele deve ser utilizado como guia absoluto para a geração de código frontend (React/Ionic).1. Estruturas Globais de Layout (Wrappers)Antes das telas específicas, o sistema possui "cascas" (Layouts) que envolvem o conteúdo dependendo da plataforma (Web vs. Mobile), conforme as Diretrizes Visuais.1.1 Wrapper Web (Desktop)Sidebar (Esquerda): Fundo #0F172A. Contém o Logotipo oficial (Gemini_Generated_Image_7s1zrk7s1zrk7s1z.png) no topo. Links de navegação: Painel, Fila do Departamento, Meu Dia, Configurações.Header (Topo): Fundo branco, altura fixa. Contém Input de Busca Global, Ícone de Notificações (Sininho) com indicador de chamados não lidos, e Avatar do Perfil.Main Content (Centro-Direita): Fundo neutro (#F8FAFC). Onde as rotas injetam as telas descritas na Seção 3.1.2 Wrapper Mobile (Ionic)Bottom Tab Bar: Navegação principal na parte inferior da tela. Abas: Painel, Fila, Criar (Botão central flutuante em Laranja #FF4500), Meu Dia, Perfil.Top App Bar: Título da tela atual e ícone de notificações. Esconde-se ao fazer scroll para baixo.2. Fluxo de Autenticação (Rotas Públicas)2.1 Tela: Login & CadastroVisual: Layout dividido (Web: Metade imagem de marca/abstrata, metade formulário; Mobile: Fundo sólido claro com formulário central). Uso do logotipo em destaque.Componentes: Inputs de E-mail e Senha (com foco destacado usando as cores da marca), Botão primário preenchido (#FF4500), Link "Esqueci minha senha".Ação: Autentica via API e guarda o JWT. Redireciona para o Fluxo de Organização.2.2 Tela: Seleção / Criação de Organização (Interstitial)Contexto: Um utilizador pode pertencer a várias empresas. Esta tela define o Tenant atual.Componentes: Lista de cartões com os logótipos e nomes das organizações às quais o utilizador pertence. Botão secundário (Outline) "Criar Nova Organização".Ação: Selecionar um cartão carrega o contexto da organização no Zustand e redireciona para a Rota Privada (Painel).3. Telas Principais da Aplicação (Rotas Privadas)3.1 Tela: Painel Geral (Dashboard)Propósito: Visão panorâmica (Métricas).Componentes:Greeting: "Olá, [Nome]" com a data de hoje.Cards de KPI: Contadores grandes -> "Em Fila" (Cinza), "Meus Pendentes" (Laranja Vivo #FF8C00), "Aguardando Revisão" (Amarelo #FFD700).Feed Recente: Lista compacta das últimas movimentações de chamados (WebSocket alimenta esta lista em tempo real).3.2 Tela: Fila do Departamento (Backlog)Propósito: Onde novas demandas chegam. Visão principal do Supervisor e do Criador.Componentes:Filtros no Topo: Tabs para "Abertos", "Em Andamento", "Cancelados".Lista de Cartões: Cada cartão mostra: Título forte, Avatar de quem criou, Prazo (em vermelho se atrasado), e uma Badge de Estado.Comportamento (Mobile): Cartões otimizados para toque. Swipe para a esquerda no cartão revela o botão "Atribuir a mim".Ação: Clicar num cartão abre a Tela: Detalhes do Chamado.3.3 Tela: Meu Dia (Cronograma / Execução)Propósito: Foco absoluto. Visão principal do Executor. Só mostra chamados atribuídos ao utilizador logado.Visual (Web): Estilo Kanban horizontal. Colunas: Para Fazer (Abertos) -> Fazendo (Em Andamento) -> Aguardando Revisão.Visual (Mobile): Lista vertical priorizada.Ações Rápidas (Drag & Drop / Swipe): O Executor pode arrastar um cartão de "Para Fazer" para "Fazendo". Isso dispara a transição de estado da FSM na API instantaneamente.3.4 Tela: Criar Novo Chamado (Modal / Overlay)Acesso: Via botão flutuante (Mobile) ou botão "Novo Chamado" no Header (Web).Componentes:Dropdown: "Para qual Departamento?" (carrega a árvore de departamentos).Inputs: Título, Descrição detalhada.Dropdown: Prioridade (Baixa, Média, Alta).Botão Submit (#FF4500).3.5 Tela: Detalhes do Chamado (Coração da FSM)Propósito: Ver informações, interagir e executar as transições da Máquina de Estados. É dividida em duas abas/painéis:Painel Esquerdo (Info): Título H1, Descrição, Metadados (Criador, Executor, Departamento, Prazo).Painel Direito (Auditoria/Chat): Timeline cronológica (estilo chat) mesclando logs de sistema ("João moveu para Em Andamento às 14:00") e comentários de texto ("Falta o arquivo anexo").Barra de Ação Fixa (Bottom/Top): Os botões visíveis aqui dependem ESTRITAMENTE do Role e do Status atual:Se Aberto e utilizador é Executor: Mostra botão "Iniciar Trabalho" (#FF8C00).Se Em Andamento e utilizador é Executor: Mostra botão "Pausar" (Outline) e "Concluir / Enviar para Revisão" (#FFD700).Se Em Revisão e utilizador é Supervisor: Mostra botões "Aprovar" (Verde Sucesso) e "Rejeitar/Devolver" (#FF4500).3.6 Tela: Organograma e Configurações (Apenas Admins/Supervisores)Propósito: Gerir subgrupos e utilizadores.Componentes:Árvore visual (Tree View) dos departamentos.Tabela de membros do departamento selecionado com Dropdown para alterar o Role (Criador <-> Executor <-> Supervisor).Botão "Gerar Link de Convite".4. Mapa de Relações (Fluxograma Lógico de Navegação)[Acesso Público]
 └── Login/Cadastro
      └──> (Sucesso) -> Seleção de Organização
                              │
[Acesso Privado - Escopo da Organização Selecionada]
 ├──> Wrapper de Layout (Carrega Web Sidebar ou Mobile Tabs)
 │
 ├──> /painel (Dashboard Inicial)
 │
 ├──> /fila (Backlog Geral)
 │     ├──> Filtra por Departamento
 │     └──> (Click Num Cartão) ───> /chamado/:id (Detalhes do Chamado)
 │                                        ├──> (Ação FSM) Atualiza Estado via API
 │                                        └──> (Chat) Adiciona Comentário
 │
 ├──> /meu-dia (Cronograma do Executor)
 │     └──> (Click Num Cartão) ───> /chamado/:id (Detalhes do Chamado)
 │
 ├──> /novo (Modal Global de Criação)
 │     └──> (Submit) -> Redireciona para /chamado/:id_novo
 │
 └──> /configuracoes (Gestão)
       └──> Gestão de Departamentos e Membros
5. Regras de Feedback de Interface (Microinterações)Ações FSM: Ao clicar num botão de mudança de estado (ex: "Iniciar Trabalho"), o botão deve exibir um spinner de carregamento e desativar-se até o backend responder.WebSockets: Se o utilizador estiver na tela /fila e um novo chamado for criado por outra pessoa, o cartão do novo chamado deve aparecer com um efeito de deslize (Slide Down) sutil e uma leve animação de brilho amarelo para chamar a atenção, sem necessidade de atualizar a página.
